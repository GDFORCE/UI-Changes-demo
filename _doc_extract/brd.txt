1. Introduction 
1.1 Document Purpose 
This Business Requirements Document (BRD) defines the complete functional and non -functional 
requirements for the Patient Visit Schedule (PVS) Mobile Application. The document serves as the 
primary reference for design, development, testing, and deployment teams involved in Phase 1 of the 
platform. 
 
1.2 Project Overview 
The Patient Visit Schedule Mobile Application is a digital platform designed to streamline the 
management of clinical trial visit schedules. The application enables Sponsors, CROs, S MOs, Sites, 
Principal Investigators (PIs), Research Teams, and Patients to coordinate, track, and manage clinical trial 
visit activities efficiently through a centralized mobile interface. 
 
1.3 Scope 
Phase 1 of the platform covers the following modules: 
• Module 1: User Registration and Login 
• Module 2: Dashboards 
• Module 3: Trial / Patient Visit Schedule Management 
• Module 4: Notifications 
• Module 5: Calendar 
• Module 6: Patient Profile & Settings 
• Module 7: Chat Interface 
• Module 8: Medication Reminder 
• Module 9: About Trial 
• Module 10: Platform Admin 
 
The mobile application backend shall be architected to support future integration with a web -based 
clinical trial ecosystem platform. 
 
1.4 Applicable Regulatory Standards 
• ICH-GCP (Good Clinical Practice) 
• NDCT Rules 2019 (New Drugs and Clinical Trials Rules, India) 
• Indian data privacy requirements

'Others Specify' Workflow 
If a user selects 'Others Specify' for any field, the system shall: 
36. Allow the user to enter a custom value 
37. Save the value for the user and send a request to the Platform Admin 
38. The Admin may Approve, Edit, or Reject the custom value 
39. If approved: the value becomes available globally to all users 
40. If edited: the corrected value is approved and made globally available 
41. If rejected: the value remains user-specific only 
This process is applicable across the entire platform wherever the 'Others Specify' option is provided.
