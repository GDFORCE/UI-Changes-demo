1. Overview & Purpose 
 
This document defines all notifications that the Platform Admin receives within the Patient Visit Schedule 
application. Admin notifications are distinct from end-user notifications — they are operational and 
compliance-driven alerts that require admin awareness or action. Every notification must be actionable, 
precise, and link directly to the screen where the issue can be resolved. 
 
PURPOSE 
Admin notifications ensure the Platform Admin is always aware of critical events, pending approvals, system 
failures, and compliance-relevant actions — without needing to manually monitor every screen. Each notification 
is a trigger for admin to act, review, or acknowledge. 
 
1.1 Admin Notification vs User Notification 
Admin notifications are a separate system from end-user notifications. They must not be mixed or confused.

2. Notification Structure — Every Notification 
 
Every admin notification — regardless of category or urgency — must contain the following fields. This 
ensures admin always has enough context to act without navigating away first. 
 
2.1 Standard Notification Fields 
# Field / element Type Mandatory? Specification & rules 
1 Notification ID Display text Auto Unique identifier for each notification instance. 
Format: ADN-XXXX (Admin Notification). 
Sequential. 
2 Category Badge / label Auto Which category triggered this notification 
(Account-Access / Organization / Trial / Issue 
etc.) 
3 Urgency level Color badge Auto Critical (dark red) / High (red) / Medium (amber) / 
Low (gray). Displayed prominently. 
4 Notification title Display text Auto Short, specific headline (max 80 characters). 
Must identify the specific event. 
5 Notification message Display text Auto Full notification content. Includes: what happened, 
who triggered it, which record is affected, what 
admin should do. Max 250 characters. 
6 Triggered by Display text Auto System-generated: 'System — Auto-detected'. 
User-generated: User name, role, entity. Admin-
generated: 'Platform Admin'. 
7 Triggered at Timestamp Auto Date, time, and timezone (IST) when the event 
occurred that triggered this notification. 
8 Related record Display text / 
link 
Auto The specific record this notification relates to: 
User ID, Trial CTRI, Ticket ID, Org ID, Subject ID 
— whichever is applicable. 
9 Navigation link Clickable link Auto Clicking the notification navigates directly to the 
screen and record where admin can act. Defined 
per notification type. 
10 Read / unread status Visual indicator Auto Unread: bold text + blue dot. Read: normal text, 
no dot. Marks as read when admin clicks. 
11 Acknowledged Optional 
checkbox 
Manual For Critical and High notifications: admin must 
explicitly acknowledge after acting. Prevents 
accidental dismissal.

5. Notification Delivery & System Rules 
 
 
5.1 Delivery Rules 
Rule Specification 
Delivery channel All admin notifications are delivered in-app to the admin portal only. No 
SMS, no email, no push notification to mobile (admin operates on web 
portal). 
Real-time delivery Notifications must appear within 30 seconds of the triggering event. No 
batching or delayed delivery acceptable. 
Critical banner persistence Critical urgency notifications show a persistent banner on all admin 
screens until acknowledged. Cannot be dismissed by scrolling or 
navigating away. 
Duplicate prevention If the same triggering event fires again before the first notification is 
acknowledged, the system updates the existing notification rather than 
creating a duplicate. 
SLA repeat notifications ADN-029 (High priority SLA breach) repeats every 60 minutes until admin 
takes action. ADN-030 (Medium priority SLA breach) repeats every 4 
hours. 
Auto-resolve behavior Notifications marked as auto-resolve=Yes disappear from the unread list 
when the underlying condition resolves (e.g., account unlocked, T&C 
accepted). 
Notification retention All notifications — read and unread — are retained in the notification 
history for 90 days. After 90 days they are archived but remain accessible 
in the audit trail. 
Audit logging Every notification delivered to admin is logged in the audit trail: notification 
ID, type, triggered at, delivered at, read at, acknowledged at (if 
applicable).
